> 版本：v0.1 MVP

> 赛事：七牛云 XEngineer AI Coding（72 小时限时）

---

## 1. 产品概述

**SynchroLens** 是一款面向中文用户的桌面 AI 同声传译助手。它实时捕获系统音频或麦克风语音，通过 STT 引擎转写为文字，再由翻译引擎生成中文译文，以悬浮字幕形式叠加在视频上方，同时自动生成带纠正标注的 Markdown 笔记。

**核心价值主张：** 让中文用户在观看英文直播、会议、课程时，无需切换窗口即可获得实时翻译字幕，并在会后获得结构化的可检索笔记。

**技术栈：** Electron + React + TypeScript + TailwindCSS + Vite  
**STT 引擎：** 讯飞 WebSocket 实时转写  
**翻译引擎：** DeepSeek V4 Pro 流式 API

## 3. 用户场景

### 场景 1：观看英文技术直播

> 张三是一名前端开发者，正在观看 React Conf 英文直播。他打开 SynchroLens，选择"系统音频"作为音源，点击开始。悬浮字幕实时显示英文原文和中文翻译，叠加在直播画面上方。直播结束后，他打开自动生成的 Markdown 笔记，回顾关键内容，发现笔记中已标注了翻译纠正（如 "hook" 被纠正为 "React Hook"）。

### 场景 2：参加英文在线会议

> 李四参加一场跨国 Zoom 会议，选择"麦克风"作为音源。SynchroLens 实时转写并翻译会议内容。会议中她注意到某句翻译不准确，通过悬浮字幕上的纠正功能修正译文。会议结束后，她触发智能总结，获得一份结构化的会议纪要。

### 场景 3：学习英文课程

> 王五在 Coursera 上学习机器学习课程，使用 SynchroLens 捕获系统音频。课程中遇到专业术语翻译不准确，他手动纠正后，后续相似语境自动应用纠正。课后笔记中，纠正处以脚注形式标注，方便他对比学习。

## 4. 功能需求

### P0 — MVP 核心（72 小时内必须完成）

#### 4.1 音频捕获

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F-P0-01 | 系统音频捕获 | 捕获系统音频输出（如浏览器、播放器声音） | 能捕获 Chrome/Edge 播放的音频，采样率 16kHz |
| F-P0-02 | 麦克风捕获 | 捕获麦克风输入 | 能捕获默认麦克风音频，采样率 16kHz |
| F-P0-03 | 音源切换 | 在系统音频和麦克风之间切换 | 切换后 1s 内生效，无需重启会话 |

#### 4.2 实时语音转写（STT）

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F-P0-04 | 讯飞 WebSocket 连接 | 建立与讯飞实时转写服务的 WebSocket 连接 | 连接成功后开始接收转写结果 |
| F-P0-05 | 逐词输出 | STT 结果以逐词（partial）形式实时输出 | 每收到一个词片段即通过 `stt:partial` 通道推送 |
| F-P0-06 | 句子确认 | STT 引擎确认句子结束时输出最终结果 | 句子确认后通过 `stt:sentence` 通道推送，携带 `sentenceId` |
| F-P0-07 | 断线重连 | 讯飞 WebSocket 断连后自动重连 | 自动重连 3 次，每次间隔 2s；重连成功后继续转写 |

#### 4.3 流式翻译

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F-P0-08 | DeepSeek 流式翻译 | 调用 DeepSeek V4 Pro 流式 API 进行翻译 | 译文以 token 粒度通过 `translate:partial` 通道推送 |
| F-P0-09 | 当前句翻译 | 对 STT 确认的句子进行翻译 | 句子翻译完成后通过 `translate:final` 通道推送 |
| F-P0-10 | 翻译超时处理 | DeepSeek API 超时时跳过当前句 | 超时阈值 10s，跳过后在字幕中显示原文无译文 |
| F-P0-11 | 限流降级 | DeepSeek API 限流时缩短上下文窗口 | 限流时上下文窗口从 5 句缩减为 1 句，避免完全中断 |

#### 4.4 当前句纠正

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F-P0-12 | 手动纠正翻译 | 用户可对当前句（未冻结）的翻译进行编辑修正 | 编辑后通过 `translate:correct` 通道推送纠正结果 |
| F-P0-13 | 纠正记录 | 记录纠正的原文、纠正后译文、纠正原因、时间戳 | 纠正记录存入 `Correction` 数据结构 |

#### 4.5 悬浮字幕窗口

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F-P0-14 | 歌词式字幕渲染 | 半透明置顶窗口，双语显示（原文+译文） | 字幕窗口始终置顶，不遮挡鼠标点击 |
| F-P0-15 | 当前句流式显示 | 当前正在翻译的句子以流式方式显示，带光标闪烁效果 | 新 token 到达时实时追加，光标在末尾闪烁 |
| F-P0-16 | 已确认句滚动 | 已确认的句子向上滚动，保留最近 5-8 句 | 超出 8 句时最早的句子淡出消失 |
| F-P0-17 | 字幕窗口拖拽 | 用户可拖拽字幕窗口到屏幕任意位置 | 拖拽后位置记忆，下次打开恢复 |
| F-P0-18 | 字幕开关 | 通过控制悬浮窗切换字幕显示/隐藏 | 隐藏时字幕窗口不显示，不影响后台转写和翻译 |

#### 4.6 控制悬浮窗

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|----------|
| F-P0-19 | 控制栏 UI | 横条状悬浮窗，包含：[▶开始/⏹停止] [字幕:开/关] [笔记:开/关] [—最小化] [×退出] | 所有按钮可点击，状态切换即时反映 |

| F-P0-22 | 退出 | 关闭应用 | 退出前若有未保存笔记则提示保存 |

#### 4.7 会话管理

| 编号 | 功能 | 描述 | 验收标准 |

## 5. 非功能需求

### 5.1 性能

| 编号 | 指标 | 目标值 | 测量方式 |
|------|------|--------|----------|
| NF-01 | 字幕延迟（端到端） | < 3 秒 | 从音频输入到字幕显示的时间差，取 P95 |
| NF-02 | STT 首字延迟 | < 500ms | 从音频片段发送到收到首个 partial 结果 |
| NF-03 | 翻译首 token 延迟 | < 1s | 从句子发送到 DeepSeek 到收到首个 token |
| NF-04 | 主进程 CPU 占用 | < 15% | 空闲时 < 5%，满载时 < 15%（不含 STT/翻译网络 I/O） |
| NF-05 | 渲染进程内存 | < 300MB | 包含主窗口 + 悬浮字幕 + 控制悬浮窗 |
| NF-06 | 笔记写入延迟 | < 1s | 从句子确认到文件写入完成 |
| NF-07 | 翻译准确率（主观评估） | ≥ 80% | 通用场景下人工评估，5 分制 ≥ 4 分 |

### 5.2 可靠性

| 编号 | 指标 | 目标值 | 实现方式 |
|------|------|--------|----------|
| NF-08 | 讯飞断线重连 | 自动重连 3 次，间隔 2s | 指数退避重连，重连成功后恢复转写 |
| NF-09 | DeepSeek 超时处理 | 超时 10s 跳过当前句 | 跳过后字幕显示原文，不阻塞后续翻译 |
| NF-10 | DeepSeek 限流降级 | 缩短上下文窗口至 1 句 | 限流时自动降级，恢复后还原 |
| NF-11 | 音频捕获失败处理 | 弹窗引导用户检查设备 | 检测到捕获失败后弹出诊断弹窗 |
| NF-12 | 笔记写入失败重试 | 重试 3 次，间隔 1s | 仍失败则弹窗提示用户手动保存 |
| NF-13 | 应用崩溃恢复 | 会话数据不丢失 | 句子数据实时持久化到文件，重启后可恢复 |

### 5.3 安全

| 编号 | 指标 | 目标值 | 实现方式 |
|------|------|--------|----------|
| NF-14 | API Key 安全存储 | 不明文存储、不写入日志 | 使用 Electron `safeStorage` 加密存储 API Key |
| NF-15 | API Key 不泄露 | 不在 IPC 通道中传输完整 Key | Main 进程持有 Key，Renderer 仅发送请求 |
| NF-16 | 网络通信加密 | STT/翻译 API 均使用 HTTPS/WSS | 讯飞 WSS、DeepSeek HTTPS |
| NF-17 | 本地笔记安全 | 笔记文件权限 600（仅当前用户可读写） | 创建文件时设置权限 |

### 5.4 可用性

| 编号 | 指标 | 目标值 | 实现方式 |
|------|------|--------|----------|
| NF-18 | 首次启动配置时间 | < 3 分钟 | 引导式配置向导：填写 API Key → 测试连接 → 开始 |
| NF-19 | 音源选择直观性 | 一键切换 | 控制悬浮窗或主窗口提供明确的音源切换按钮 |
| NF-20 | 字幕可读性 | 半透明背景 + 合理字号 | 字幕背景透明度 70%，字号 ≥ 18px，支持用户调节 |
| NF-21 | 错误提示友好性 | 所有错误均给出用户可操作的提示 | 不展示技术错误码，提供"重试"/"检查设置"等操作引导 |

## 6. 竞品分析

| 维度 | SynchroLens | 实时字幕（Chrome 内置） | 谷歌翻译 App | 讯飞听见 | 同传软件（如 Interact） |
|------|----------|------------------------|-------------|----------|------------------------|
| **实时性** | < 3s 端到端延迟 | 1-2s（仅 Chrome 标签页） | 2-4s | 2-3s | 1-3s |
| **翻译质量** | DeepSeek V4 Pro，支持纠正 | Chrome 内置翻译，不可纠正 | Google NMT，不可纠正 | 仅转写无翻译 | 依赖引擎，多数不可纠正 |
| **字幕叠加** | 悬浮置顶，歌词式，可拖拽 | 仅限 Chrome 标签内 | 全屏悬浮，不可拖拽 | 无悬浮字幕 | 部分支持悬浮 |
| **笔记生成** | 自动 Markdown + 纠正脚注 | 无 | 无 | 导出 SRT/TXT | 部分支持导出 |
| **音源支持** | 系统音频 + 麦克风 | 仅 Chrome 标签页 | 麦克风 | 麦克风/文件 | 麦克风/系统音频 |
| **纠正能力** | 当前句手动纠正 + 批量检测 | 无 | 无 | 无 | 无 |
| **桌面端** | ✅ Electron 原生 | ❌ 浏览器扩展 | ❌ 移动端为主 | ✅ Web/客户端 | ✅ 桌面端 |
| **离线能力** | ❌ 需联网 | ✅ 部分支持 | ✅ 部分支持 | ❌ 需联网 | ❌ 需联网 |

**SynchroLens 差异化优势：**
1. **纠正闭环**：唯一支持"当前句纠正 + 笔记纠正脚注"的产品，翻译质量可持续改善
2. **笔记一体化**：从字幕到笔记到总结，一站式产出，无需二次整理
3. **系统音频捕获**：不局限于浏览器标签页，可覆盖任意桌面应用音频
4. **悬浮字幕体验**：歌词式流式渲染 + 可拖拽，视觉体验优于竞品

## 附录 A：数据结构定义

```typescript
/** STT 转写结果 */
interface STTResult {
  sentenceId: string;   // 句子唯一标识
  text: string;         // 转写文本
  isFinal: boolean;     // 是否为最终确认结果
  timestamp: number;    // 时间戳（ms）
}

/** 翻译结果 */
interface TranslationResult {
  sentenceId: string;           // 句子唯一标识
  original: string;            // 原文
  translation: string;         // 译文
  isFinal: boolean;            // 是否为最终确认结果
  corrections: Correction[];   // 纠正记录
}

/** 纠正记录 */
interface Correction {
  from: string;         // 纠正前译文
  to: string;           // 纠正后译文
  reason: string;       // 纠正原因（用户手动 / 批量检测）
  timestamp: number;    // 纠正时间戳（ms）
}

/** 会话 */
interface Session {
  id: string;                   // 会话唯一标识
  startTime: number;            // 开始时间戳（ms）
  endTime?: number;             // 结束时间戳（ms）
  audioSource: 'system' | 'mic'; // 音源类型
  sentences: TranslationResult[]; // 所有句子
  notePath?: string;            // 笔记文件路径
  summary?: string;             // 智能总结
}
```

## 附录 B：IPC 通道一览

| 通道 | 方向 | 载荷 | 说明 |
|------|------|------|------|
| `stt:partial` | Main → Renderer | `STTResult` | STT 逐词输出 |
| `stt:sentence` | Main → Renderer | `STTResult` | STT 句子确认 |
| `translate:partial` | Main → Renderer | `TranslationResult` | 翻译逐 token 输出 |
| `translate:final` | Main → Renderer | `TranslationResult` | 翻译完成 |
| `translate:correct` | Main → Renderer | `TranslationResult` | 翻译纠正 |
| `note:saved` | Main → Renderer | `{ notePath }` | 笔记保存成功 |
| `note:summary` | Main → Renderer | `{ summary }` | 总结生成完成 |
| `session:start` | Renderer → Main | `{ audioSource }` | 开始会话 |
| `session:stop` | Renderer → Main | `{}` | 停止会话 |
| `session:pause` | Renderer → Main | `{}` | 暂停会话 |
| `config:update` | Renderer → Main | `Config` | 更新配置 |
| `summary:trigger` | Renderer → Main | `{}` | 触发总结 |
