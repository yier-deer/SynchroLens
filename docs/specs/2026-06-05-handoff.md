## 7. 关键约束和教训（从前两个会话总结）

### ⚠️ 子 Agent 调度失败根因（不要重蹈覆辙）

| 根因 | 现象 | 正确做法 |
|------|------|----------|
| 子 Agent prompt 过大 | 执行中断/超时 | 主会话直接写代码 |
| 子 Agent 内执行 `npm install` / `npx jest` | 沙箱超时 | 主会话统一负责 |
| 多个 Agent 互相等待基础设施 | 死锁 | 基础设施由 Layer 1 统一完成 |
| **PR 粒度太大**（24 文件 / PR） | 虽然已合并但不符比赛要求 | 每个 PR ≤ 3 文件 |

### 📁 测试文件路径深度

```
Test/shared/   → ../../src/shared/          （2 级）
Test/preload/  → ../../src/preload/         （2 级）
Test/main/utils/    → ../../../src/main/utils/    （3 级）
Test/main/ipc/      → ../../../src/main/ipc/       （3 级）
Test/main/modules/  → ../../../src/main/modules/   （3 级）
```

### 🔒 类型导入方式

- `src/main/` 模块导入 `src/shared/`：使用 `import { ... } from '../../../shared/xxx'`（相对路径）
- `src/renderer/` 使用 `@shared/` 别名（由 electron.vite + tsconfig.web.json 配置）

### 📝 编码规范（详见 CONVENTIONS.md）

- 中文 JSDoc 注释（所有函数、方法、类）
- 2 空格缩进，单引号，分号必加
- 类型守卫函数名 `isXxx`
- 布尔函数 `is`/`has`/`should` 开头
- 事件处理 `handle` 开头
- 命名导出，禁止默认导出
